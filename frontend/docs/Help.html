API Reference

REST Endpoints

All API endpoints are served by the Express server at http://localhost:3000 (or the configured PORT). The server also serves frontend static files from the same origin.

Health check
GET  /health
# Returns: { "status": "ok", "db": "connected", "uptime": 42.1 }

Networks API

/api/networks
GET /api/networks List all available networks ▶

Returns a sorted list of all ingested network source filenames.

Response 200
[
  "O52K1_HUMAN_Example1.csv",
  "METN_ECOLI_Example2.csv",
  "my_proteins.csv"
]

Returns 500 if the database is unavailable.

GET /api/networks/:filename Fetch nodes + edges for one network ▶
ParamTypeDescription
:filenamestringrequiredURL-encoded network filename (e.g. my_proteins.csv)
Response 200
{
  "elements": {
    "nodes": [
      { "data": { "id": "P12345", "NH_ID": "NH001", "NH_Size": 14, "NCBI_txID": "9606" } }
    ],
    "edges": [
      { "data": { "id": "P12345|Q67890", "source": "P12345", "target": "Q67890", "weight": 0.854 } }
    ]
  }
}

Returns 404 if the filename is not found. Edge IDs are canonical: smaller_id|larger_id (alphabetically sorted).

POST /api/networks/search Find nodes by UniProt accession ▶

Searches the specified network for proteins matching exact UniProt accession numbers. Results are scoped strictly to the given network — accessions in other networks are not returned.

Request body
{
  "network":     "my_proteins.csv",
  "accessions": ["P04637", "O15151"]
}
Response 200
{
  "matches": [
    { "id": "P04637", "nh_id": "NH001" },
    { "id": "O15151", "nh_id": "NH001" }
  ]
}

nh_id is null if the protein has no neighborhood assignment in the attributes file.

POST /api/networks/search-species Find nodes by NCBI taxonomy ID ▶
Request body
{
  "network":    "my_proteins.csv",
  "speciesIds": ["9606", "10090"]
}
Response 200
{
  "matches": [
    { "id": "P04637", "nh_id": "NH001" },
    { "id": "A0A2I3TW31", "nh_id": "NH004" }
  ]
}
POST /api/networks/group-exports Save named protein groups to disk ▶

Writes one text file per group to the configured EXPORTS_PATH directory (data/exports/ by default).

Request body
{
  "groups": [
    { "name": "GroupA", "accessions": "P04637 O15151 Q13418" },
    { "name": "GroupB", "accessions": "P98765 A0A2I3TW31" }
  ]
}
Response 200
{
  "savedFiles": ["GroupA.txt", "GroupB.txt"],
  "exportDir":  "data/exports"
}

Group names allow only letters, digits, _, -, and .. Maximum 16 characters. Names must be unique within a request.

Subnetworks API

/api/subnetworks
GET /api/subnetworks/limits Return extraction limits and available indexes ▶
Response 200
{
  "maxSeeds":        10,
  "maxNodes":        2500,
  "defaultMaxNodes": 500,
  "indexes": [
    { "label": "Eukaryotes", "value": "Eukaryotes" },
    { "label": "Bacteria",    "value": "Bacteria" }
  ]
}
POST /api/subnetworks Extract a subnetwork from the binary index ▶

Spawns the Python extraction script, writes the resulting CSV to data/networks/, and returns a viewer link. The file watcher auto-ingests the new CSV.

FieldTypeDescription
namestringrequiredOutput filename (without .csv). Max 80 chars. Pattern: [A-Za-z0-9_.-]+
indexstringrequiredIndex prefix, e.g. Eukaryotes or Bacteria
seedsstringrequiredComma/space-separated UniProt ACs. Max 10 seeds, total length ≤ 2000 chars
maxNodesintegeroptionalNode budget. Default 500, max 2500
Response 200
{
  "ok":           true,
  "viewerUrl":    "/viewer.html?network=my_proteins.csv&seeds=P04637",
  "edgeCount":    1284,
  "selectedNodeCount": 498,
  "missingSeeds": [],
  "elapsedMs":    3241
}

Returns 400 for validation errors and 408 on timeout (default 120 s, configurable via SUBNETWORK_TIMEOUT_MS).

Species API

/api/species-names · /api/species-tree
GET /api/species-names NCBI taxonomy ID → species name mapping ▶

Reads NCBI_txID.csv on every request and returns all rows. Used by the species selector flat-list fallback.

Response 200
[
  { "ncbi_txid": "9606",  "species_name": "Homo sapiens" },
  { "ncbi_txid": "10090", "species_name": "Mus musculus" }
]
GET /api/species-tree Full parsed taxonomic tree (cached) ▶

Returns the parsed commontree.txt as a nested JSON tree, cached in memory. Cache is invalidated when taxonomy files change on disk.

Returns 404 if commontree.txt is absent — the frontend falls back to the flat species list.

UniProt API

/api/uniprot
GET /api/uniprot/:accession Proxied UniProt REST entry ▶

Proxies a request to https://rest.uniprot.org/uniprotkb/:accession with a 10 s timeout and up to 3 retries. The tooltip module calls UniProt directly from the browser — this proxy is for server-side use.

POST /api/uniprot/availability Bulk check UniProt entry existence ▶

Used by the Export panel to decide whether to render accessions as hyperlinks. Checks up to 100 accessions per call in batches.

Request body
{ "accessions": ["P04637", "FAKE9999"] }
Response 200
{
  "results": [
    { "accession": "P04637",  "available": true  },
    { "accession": "FAKE9999", "available": false }
  ]
}

Startup Modes

START_MODE env variable · backend/entrypoint.js

Set START_MODE in .env to control what happens at startup:

Modenpm scriptBehavior
servenpm startDefault. Auto-ingests from scratch if the database is empty (first run), then starts the server. Safe to re-run at any time.
ingestnpm run ingestRuns the full ingestion pipeline (attributes + all CSVs) then exits. No server is started.
ingest-and-servenpm run start:ingest-and-serveIngests, then starts the server. Use when you have new data files and want to guarantee a fresh load.
First-run detection

In serve mode, the entrypoint queries SELECT COUNT(*) FROM nodes. If it returns 0, full ingestion runs before the server starts. This is how the first npm start works without needing a separate ingest command.

Network CSV Format

data/networks/*.csv

Each line defines one undirected edge. No header row.

Format
# node1,node2,weight
P12345,Q67890,0.854231
P12345,A0A0H0PPY1,0.712045
ColumnTypeRules
node1stringUniProt accession (or any string ID). Must not be blank.
node2stringSame. Must differ from node1 (self-loops produce no visible edge).
weightfloatMust be a finite number. Rows with non-numeric or infinite weights are silently skipped.
Edge ID canonicalization

Edge IDs are stored as min(node1,node2)|max(node1,node2) (alphabetical sort). The order of columns in the CSV does not matter — both A|B and B|A resolve to the same edge.

Node Attributes File Format

data/nodes_attr/*.nodes.attr

Comma-separated, with header row. Exactly one file must be present in the directory.

Format
node_id,NCBI_txID,NH_ID,NH_Size
P12345,9606,NH001,14
Q67890,9606,NH001,14
A0A0H0PPY1,10090,NH002,7
ColumnRequiredDescription
node_idYesUniProt accession. Must be globally unique in the file.
NCBI_txIDYesNCBI taxonomy ID. Used by the species filter.
NH_IDYesNeighborhood cluster ID. Proteins sharing the same NH_ID form a cluster.
NH_SizeYesTotal number of proteins in this cluster (used for ProtDC normalization).
reserved1–5NoExtra columns are silently ignored (stripped from stored attributes).
Constraint — one file, unique accessions

The directory must contain exactly one .nodes.attr file. Every node_id must be unique — duplicates cause ingestion to abort with an error naming both line numbers and both NH_IDs.

Binary Graph Index Format

data/indexes/ — used by extract_subnetwork.py

Three files per domain form one index. The prefix (e.g. Eukaryotes) must match the value presented in the domain dropdown.

FileDescription
*.node_ids.tsvTwo-column tab-separated: integer_id\tUniProt_AC. Maps internal integer IDs ↔ accession strings.
*.adj.index.binBinary index header + per-node (offset, degree) entries. Header: 4-byte uint32 node count. Entries: 12 bytes each (8-byte uint64 offset into adj.bin, 4-byte uint32 degree).
*.adj.binFlat binary adjacency list. Each entry is 8 bytes: 4-byte uint32 neighbor ID + 4-byte float32 weight. Neighbors are pre-sorted by descending weight.

These files are memory-mapped at extraction time via Python's mmap and read with NumPy for speed. They are read-only by the application.

Taxonomy Files

data/NCBI_txID/

NCBI_txID.csv

Two-column CSV with header. Used both for the flat species dropdown and as the label source for the taxonomic tree.

Format
ncbi_txid,species_name

10090,Mus musculus
7227,Drosophila melanogaster

commontree.txt

ASCII tree downloaded from the NCBI Common Tree tool. The parser extracts taxon IDs from lines of the form:

    Metazoa
      Chordata
        Mammalia

            9606: Homo sapiens

Both files are hot-reloaded — replacing them on disk while the server is running invalidates the cached tree within about 2 seconds.

Backend Internals

Ingestion pipeline · file watcher · query layer

Ingestion Pipeline (ingestData.js)

Two independent pipelines share the same module:

ingestNodeAttributes(files) — Transactional, all-or-nothing:

  1. Preflight — validates header format; scans for duplicate UniProt ACs across all files (reports file, line, NH_ID for each conflict).
  2. Reconcile — inside a single BEGIN TRANSACTION, clears attributes_json and attribute_source for every protein row that has non-empty attributes. Also deletes unreferenced kind='nh' rows (one-time migration).
  3. Write — upserts each row using INSERT … ON CONFLICT DO UPDATE. Batches use SAVEPOINT/RELEASE for memory management without breaking outer atomicity.
  4. Commit or rollback — any failure in any file triggers ROLLBACK, restoring the entire pre-ingest state.

ingestNetworks(specificFile) — Per-file, serialized via a promise queue:

  1. Deletes all edges for the source file (DELETE … WHERE source = ?).
  2. Inserts placeholder nodes (INSERT OR IGNORE) before edges to satisfy foreign key constraints.
  3. Inserts edges with a SAVEPOINT-based batch loop.
  4. Skips rows with fewer than 3 columns, blank node IDs, or non-finite weights.

File Watcher (fileWatcher.js)

Uses Chokidar with polling (interval: 1000 ms, stabilityThreshold: 2000 ms) for compatibility across filesystems and Docker mounts.

  • *.csv added/changed → ingestNetworks(filename) for that file only.
  • *.csv deleted → removes edges and network_nodes for that source.
  • *.nodes.attr changed → re-scans directory; ingests if exactly one file is present.
  • Taxonomy files changed → invalidates the in-memory species tree cache.
  • Internally-saved networks (from subnetwork extraction or edit service) are suppressed for up to 300 s / 3 events to avoid double-ingest.

Frontend Internals

Graph data model · visible graph · D3 rendering

Graph (core/graph.js)

Framework-agnostic graph data structure. Stores nodes in a Map<id, node>, edges in a Map<id, edge>, and adjacency as a Map<nodeId, Set<edgeId>>. Never modified by the rendering layer.

GraphView (core/graph-view.js) — Collapsed vs. Expanded

GraphView.compute(coreGraph, expandedClusters, hiddenNodes, hiddenEdges) derives the visible graph on each render:

  1. For each visible protein: if its NH_ID is not in expandedClusters, its representative is the cluster node. Otherwise, it represents itself.
  2. Cluster nodes are created on the fly with kind: 'nh', size = member count, _memberIds list.
  3. Edges between the same representative are discarded (self-loops). Inter-representative edges accumulate weight and are normalized: sum(weights) / (size_u × size_v).

Cluster nodes are never stored in the database. They are purely a render-time derivation.

AppState (core/state.js)

Single source of truth for all application state. Key fields:

  • graph — the full core Graph (all proteins + edges from the loaded network)
  • viewGraph — the currently rendered Graph (computed by GraphView)
  • expandedClusters — Set of NH IDs that are currently expanded
  • highlightLayers — Map of layerId → { matches, color }
  • nodeColors — derived Map of nodeId → [color, …] (pie slices)
  • hiddenNodes / hiddenEdges — Sets of hidden element IDs
  • hiddenEdgeWeightRanges — compact threshold rules for weight-based filtering
  • selectedNodes — Set of selected node IDs (export mode)
  • topologyRevision — integer incremented on any structural change (used as cache key)

D3Adapter (adapters/d3-adapter.js)

Bridges AppState to D3.js. Key behaviors:

  • Static layout — runs the force simulation for a fixed tick count scaled to sqrt(nodeCount), then freezes it. No continuous animation.
  • Position preservation — on topology changes (expand/collapse), existing node positions are carried forward. New cluster members are seeded in a golden-angle spiral around their parent.
  • Component layout — disconnected graph components are packed into non-overlapping circles using an iterative packing algorithm, then each component's centroid is used as its force-center attractor.
  • Pinned nodes — drag sets node.fx / node.fy. Pinned positions survive topology changes but are cleared on a fresh network load.
  • Viewport-aware fit — fitToView() accounts for occluding UI panels (zoom controls, export controls) when computing the usable viewport bounds.

Module System

core/module-system.js · config/modules.js

Modules are plain objects registered with ModuleSystem.register(module). They receive a context object on init that exposes safe AppState methods — modules never touch AppState directly.

Adding a Module

  1. Create frontend/js/modules/my-feature.js exporting window.MyFeatureModule with an id string and an init(context) function.
  2. Add "my-feature" to the activeModules array in frontend/js/config/modules.js.
  3. Add the script tag to viewer.html. No core files need to change.
species-selector
Species Selector

Floating panel with interactive taxonomic tree (or flat checkbox fallback). Queries the backend for proteins by NCBI_txID and adds a highlight layer.

search-highlight
Search & Highlight

Accession-based search with canvas color picker. Creates named highlight layers with per-layer color; supports multiple simultaneous layers.

clear-highlights
Clear Highlights

Single-button module that calls appState.clearHighlightLayers(). Removes all color layers and resets node appearance.

uniprot-tooltip
UniProt Tooltip

Hover tooltip fetching protein name and organism from UniProt REST API. For clusters, samples up to 5 random members to find one with a function annotation.

export-panel
Export Panel

Selection mode, protein export sections, ProtDC scoring, UniProt link management, and Batch & Comparative Analysis popup with group file saving.

Event Bus

AppState.eventBus — CustomEvent-based pub/sub

Components communicate by emitting and listening to events on AppState.eventBus (a native EventTarget). Use appState.on(event, callback) to subscribe and appState.emit(event, detail) to publish.

  • graphUpdatedFired on network load, expand/collapse, or any edit. Detail: { layoutReset, fitView, preservePins, topologyRevision, reason }. D3Adapter re-renders.
  • graphVisualsUpdatedFired when only colors change (no topology change). D3Adapter re-draws node visuals only.
  • clusterExpandedFired when a single cluster is expanded. Detail: { clusterId, topologyRevision }.
  • clusterCollapsedFired when a single cluster is collapsed. Detail: { clusterId, topologyRevision }.
  • editUpdatedFired after hide/show node or edge operations. Detail: edit stats object (totalProteinCount, visibleProteinCount, etc.).
  • selectionModeChangedFired when export selection mode is toggled. Detail: boolean. D3Adapter enables/disables brush.
  • selectionUpdatedFired when the selected node set changes. Detail: Array<nodeId>.
  • nodeHoverFired by D3Adapter on mouseover/mousemove/mouseout. Detail: { nodeId, data, x, y, type }. UniprotTooltipModule listens.
  • viewGraphUpdatedFired after AppState.setViewGraph(). Carries edit stats.

Database Schema

SQLite — data/network_viz.db
SQL — nodes table
CREATE TABLE nodes (
  id               TEXT PRIMARY KEY,
  kind             TEXT NOT NULL,       -- always 'protein' (NH nodes never stored)
  attributes_json  TEXT,              -- JSON object; '{}' = placeholder (no attr file row)
  attribute_source TEXT               -- filename that wrote this row's attributes
);
SQL — edges table
CREATE TABLE edges (
  id              TEXT NOT NULL,         -- canonical: sorted(node1, node2) joined with |
  node1           TEXT NOT NULL,
  node2           TEXT NOT NULL,
  weight          REAL,
  source          TEXT NOT NULL,         -- source CSV filename
  attributes_json TEXT,
  PRIMARY KEY (id, source),
  FOREIGN KEY(node1) REFERENCES nodes(id),
  FOREIGN KEY(node2) REFERENCES nodes(id)
);
SQL — network_nodes table
CREATE TABLE network_nodes (
  source   TEXT NOT NULL,   -- source CSV filename
  node_id  TEXT NOT NULL,   -- node that belongs to this network
  PRIMARY KEY (source, node_id)
);

attribute_source on the nodes table is added via ALTER TABLE migration for databases created before it existed. The column enables correct reconcile (stale legacy rows with attribute_source = NULL are cleared on next ingestion).

Why network_nodes?

Isolated nodes (present in a network but with no edges) cannot be discovered from the edges table alone. The network_nodes table records every node explicitly registered to a source, regardless of degree.
